# 개발로그 운영 규칙

## 목적
- 단계별 진행 상황을 명확하게 기록
- 체크리스트 항목 완료 시점의 근거를 남김
- GitHub 공유 시 변경 의도를 빠르게 파악 가능

## 작성 규칙
1. 파일명: `YYYY-MM-DD-XX-주제.md`
2. 각 로그에 최소 포함
   - 목표
   - 완료 체크리스트
   - 구현 내용
   - 검증 결과
   - 다음 단계
3. 체크리스트 항목 완료 즉시 로그 추가
4. 이슈/리스크가 있으면 별도 섹션으로 명시

## 템플릿
```md
# YYYY-MM-DD / 단계명

## 목표
- ...

## 완료 체크리스트
- [x] ...
- [ ] ...

## 구현 내용
- ...

## 검증 결과
- ...

## 리스크/메모
- ...

## 다음 단계
- ...
```
